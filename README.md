# 🚀 Git & GitHub Workshop

Welcome! This repo is designed to help you learn Git and GitHub from scratch. No experience needed — just follow along, and by the end, you'll have made your very first pull request!

## What is Git?

**Git** is a tool that helps you track changes in your code. Think of it like a save system for your projects — you can save different versions, go back to an older version, and work on different features without messing up the main project. It runs on your computer and works completely offline.

## What is GitHub?

**GitHub** is a website that stores your Git projects online. It makes it easy to share your code with others, work together on the same project, and show off your work. Think of it as social media for code.

**In short:** Git is the tool, GitHub is the website.

## Getting Started

### 1. Create a GitHub Account

If you don't have a GitHub account yet, head over to [github.com/signup](https://github.com/signup) and create one. It's free!

> Pick a good username — this will be part of your profile link (github.com/your-username).

### 2. Install Git

You need to install Git on your computer:

- **Windows:** Download from [git-scm.com/downloads/win](https://git-scm.com/downloads/win) and run the installer. Use the default settings.
- **macOS:** Open Terminal and run:
  ```
  xcode-select --install
  ```
  Or download from [git-scm.com/downloads/mac](https://git-scm.com/downloads/mac).
- **Linux:** Open your terminal and run:
  ```
  sudo apt install git
  ```
  (For other Linux distros, check [git-scm.com/downloads/linux](https://git-scm.com/downloads/linux))

### 3. Verify the Installation

Open your terminal (or Git Bash on Windows) and run:

```
git --version
```

You should see something like `git version 2.x.x`. If you do, you're good to go!

### 4. Configure Git

Tell Git who you are. This information will be attached to your contributions:

```
git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"
```

> Use the same email address you used to create your GitHub account.

## Key Concepts

Before jumping into the challenge, here are some important terms you'll come across:

| Term | What it means |
|------|---------------|
| **Repository (repo)** | A project folder tracked by Git. It contains all your files and the history of changes. |
| **Clone** | Making a copy of a repository from GitHub to your computer. |
| **Fork** | Making your own copy of someone else's repository on GitHub (under your account). |
| **Branch** | A separate version of the repo where you can make changes without affecting the main version. |
| **Commit** | A saved snapshot of your changes, like a checkpoint in a game. |
| **Push** | Uploading your commits from your computer to GitHub. |
| **Pull Request (PR)** | A request to merge your changes into the original repository. This is how you contribute to other people's projects! |

## 🏆 The Challenge

Ready to put what you've learned into action? Here's your challenge:

1. **Fork** this repository to your own GitHub account
2. **Clone** your fork to your computer
3. **Create a new branch** for your changes
4. **Add your name** to the `names.txt` file
5. **Commit** your changes
6. **Push** your branch to your fork on GitHub
7. **Open a pull request** to this repository
8. 🎉 Check the [Workshop Board](https://fawazabdullah25.github.io/git-workshop/) to see your name on the board!

> **Stuck?** Don't worry! Check out the [detailed instructions](instructions.md) for a step-by-step walkthrough with all the commands you need.
